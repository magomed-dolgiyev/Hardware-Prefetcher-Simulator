The 4 Prefetchers

🚫

No Prefetcher

Baseline. Zero prefetching. Every access is a potential cache miss.

Baseline
➡️

Next-Line

After each access fetches the immediately next cache line (addr + 64).

Best for sequential streams. Fails on strides > 64 B.

Simple · Low cost
📏

Stride

Tracks per-PC access history. Detects repeated stride, prefetches N lines ahead.

Confirm threshold = 2. Works for any regular stride pattern.

PC-indexed · degree=4
🌊

GHB / Stream

Global History Buffer — delta correlation indexed by PC. Handles multiple concurrent memory streams.

Lookback=3, history=256 entries.

Multi-stream · Advanced

Experiment 1

Cache Hit Rate by Trace Type

  • 🔵Sequential — Next-Line wins at 100%. Stride/GHB at 50% (every 2nd line is a "next" line, so they overshoot).
  • 🟠Strided-128 — Next-Line collapses (stride 128 ≠ 64 B). Stride & GHB hold at ~49%.
  • 🟢Mixed — Stride & GHB lead at 57%. Next-Line manages 27%.
  • ❌Random — Everyone fails (~3%). Unpredictable addresses are unbeatable.
  • ✅Looping — All prefetchers ~100%: tiny working set fits in 32 KB cache.
Experiments 2 & 3

Metrics vs Cache Size & Accuracy/Coverage

Next-Line dominates on Mixed trace at all cache sizes (82% hit rate, ~82% accuracy). Stride accuracy jumps to 50% only at 64 KB+ — needs large cache to "stage" early prefetches.

Top-right = ideal zone. Next-Line on Sequential (100%/100%) is closest to ideal. All prefetchers collapse on Random (bottom-left). Stride & GHB cluster at ~20% accuracy / ~50% coverage for predictable traces.

Experiment 5 · Conclusions

Degree Sensitivity & Key Findings

Degree=1 gives best accuracy (100%), but hit rate drops 50% at degree≥2 because the cache evicts pre-fetched lines before they're used on the Mixed trace.

  • 🏆No single prefetcher wins everywhere — access pattern matters most.
  • 🔵Next-Line is optimal for sequential workloads; useless for stride-128.
  • ⚖️Stride & GHB are more general: consistent 49–57% hit rate on structured traces.
  • 📐Prefetch degree=1 maximises accuracy; higher degree increases coverage but pollutes cache.
  • 🎲Random access (~3%) is unpredict­able for all hardware prefetchers — need software hints.
  • 🔁Looping workloads saturate the cache naturally; prefetching adds little extra benefit.
